## Stocker du texte encodé en UTF-8 avec des chaînes de caractères

Nous avons parlé des chaînes de caractères dans la [section sur la propriété](course://Understanding Ownership/What is ownership/The String Type), mais nous allons les examiner plus en profondeur maintenant. Les nouveaux utilisateurs de Rust sont souvent bloqués sur les chaînes de caractères pour une combinaison de trois raisons : la propension de Rust à exposer les erreurs possibles, les chaînes de caractères étant une structure de données plus complexe que ce que de nombreux programmeurs leur attribuent, et l’UTF-8. Ces facteurs se combinent de manière à ce que cela puisse sembler difficile lorsque vous venez d'autres langages de programmation.

Il est utile de discuter des chaînes de caractères dans le contexte des collections, car les chaînes de caractères sont implémentées comme une collection d'octets, plus certaines méthodes qui offrent une fonctionnalité utile lorsque ces octets sont interprétés comme du texte. Dans cette section, nous parlerons des opérations sur `String` que chaque type de collection possède, telles que la création, la mise à jour et la lecture. Nous discuterons également des différences entre `String` et les autres collections, notamment la complexité de l'indexation dans une `String` due aux différences entre la manière dont les gens et les ordinateurs interprètent les données de `String`.

### Qu'est-ce qu'une chaîne de caractères ?

Nous définirons d’abord ce que nous entendons par le terme _chaîne de caractères_. Rust n’a qu’un seul type de chaîne de caractères dans son langage de base, qui est la tranche de chaîne `str`, généralement vue sous sa forme empruntée `&str`. Dans le chapitre "Comprendre la propriété", nous avons parlé des _tranches de chaîne_, qui sont des références à des données de chaîne encodées en UTF-8 stockées ailleurs. Les littéraux de chaîne, par exemple, sont stockés dans le binaire du programme et sont donc des tranches de chaîne.

Le type `String`, qui est fourni par la bibliothèque standard de Rust plutôt que codé dans le langage de base, est un type de chaîne de caractères propre, mutable et encodée en UTF-8. Lorsque les Rustaceans parlent de « chaînes de caractères » en Rust, ils se réfèrent généralement aux types `String` et `&str`, pas seulement à l'un de ces types. Bien que cette section concerne largement `String`, les deux types sont largement utilisés dans la bibliothèque standard de Rust, et les deux `String` et tranches de chaînes sont encodés en UTF-8.

La bibliothèque standard de Rust inclut également un certain nombre d'autres types de chaînes, tels que `OsString`, `OsStr`, `CString` et `CStr`. Les bibliothèques externes peuvent offrir encore plus d'options pour stocker des données de chaîne. Voyez comment ces noms se terminent tous par `String` ou `Str` ? Ils se réfèrent aux variantes possédées et empruntées, tout comme les types `String` et `str` que vous avez déjà vus. Ces types de chaînes peuvent stocker du texte dans différents encodages ou être représentés en mémoire d'une manière différente, par exemple. Nous ne discuterons pas de ces autres types de chaînes dans ce chapitre ; consultez leur documentation API pour en savoir plus sur comment les utiliser et quand chacun est approprié.
