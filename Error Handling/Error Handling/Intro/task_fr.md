## Gestion des erreurs

L'engagement de Rust envers la fiabilité s'étend à la gestion des erreurs. Les erreurs sont une réalité dans le logiciel, donc Rust possède un certain nombre de fonctionnalités pour gérer les situations où quelque chose se passe mal. Dans de nombreux cas, Rust vous oblige à reconnaître la possibilité d'une erreur et à prendre des mesures avant que votre code puisse être compilé. Cette exigence rend votre programme plus robuste en garantissant que vous découvrirez les erreurs et les traiterez de manière appropriée avant d'avoir déployé votre code en production !

Rust regroupe les erreurs en deux grandes catégories : les erreurs _récupérables_ et les erreurs _irrécupérables_. Pour une erreur récupérable, comme une erreur de fichier introuvable, il est raisonnable de signaler le problème à l'utilisateur et de réessayer l'opération. Les erreurs irrécupérables sont toujours des symptômes de bogues, comme tenter d'accéder à un emplacement au-delà de la fin d'un tableau.

La plupart des langages ne distinguent pas ces deux types d'erreurs et les gèrent de la même manière, en utilisant des mécanismes tels que les exceptions. Rust n'a pas d'exceptions. À la place, il dispose du type `Result<T, E>` pour les erreurs récupérables et de la macro `panic!` qui arrête l'exécution lorsque le programme rencontre une erreur irrécupérable. Ce chapitre couvre d'abord l'appel de `panic!` puis discute de la renvoi de valeurs `Result<T, E>`. De plus, nous explorerons les considérations à prendre en compte pour décider s'il faut tenter de récupérer d'une erreur ou arrêter l'exécution.
